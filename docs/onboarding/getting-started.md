# Level 0 — 뭔지 일단 감 잡기

> **유형**: Tutorial · **독자**: Level 0 · **읽는 시간**: ~3분

여기는 "이 레포를 써야 하나 말아야 하나" 를 빠르게 가늠해 보는 곳이에요. 코드를 한 줄도 돌려보지 않아도, 아직 클론조차 안 했어도 읽을 수 있게 쓰여 있어요. Spring Boot 가 처음이어도 괜찮아요. 지금 필요한 건 설치가 아니라 "이게 내 상황에 맞는 도구인가" 라는 판단 하나거든요.

그 판단을 3분 안에 내릴 수 있게, 옆에서 한 단계씩 안내할게요.

## 누가 읽으면 좋은가

이 레포 링크를 처음 받아 든 사람을 위한 글이에요. 다음 중 하나라도 해당하면 잘 찾아왔어요.

- "Spring 백엔드 템플릿" 이 뭔지 아직 감이 없는 사람
- 비슷한 걸 찾고 있던 개발자, 또는 팀의 PM·디자이너·프론트엔드
- 시간을 투자해 더 깊이 볼지 아직 결정하지 못한 사람

## 이 레벨의 목표

3분에서 길어도 10분 안에, 다음 세 질문에 스스로 답할 수 있게 만드는 게 목표예요.

1. 이게 뭔가 — 한 줄로 설명할 수 있다
2. 내 상황에 맞는가 — 쓴다·안 쓴다·고민된다 중 하나로 판단할 수 있다
3. 그다음 뭘 봐야 하나 — 다음 문서로 자연스럽게 넘어갈 수 있다

반대로, 이 레벨이 **하지 않는 일** 도 분명히 해 둘게요. 여기서 실제로 앱을 돌려보거나, 구조를 속속들이 이해하거나, 설계 결정의 이유까지 파고들 필요는 없어요. 그건 각각 다음 문서가 맡아요.

- 직접 돌려보기 — [`템플릿 첫 사용 가이드`](../start/onboarding.md) 가 노트북에 환경을 띄우는 과정을 안내해요.
- 구조 전체 파악 — [`Architecture Reference`](../structure/architecture.md) 가 모듈과 기술 스택을 정리해요.
- 설계 이유 파고들기 — [`Repository Philosophy`](../philosophy/README.md) 가 38개 ADR 로 "왜 이렇게 만들었나" 를 풀어요.

## 무엇부터 읽을까

Level 0 문서는 가벼운 것부터 손에 잡는 것 순으로 놓여 있어요. 위에서 둘만 읽어도 "쓸지 말지" 결정에는 충분해요. 나머지는 마음이 기울었을 때 이어 가면 돼요.

먼저, 코드를 돌리지 않고 읽는 두 편이에요.

1. [`이게 뭐야?`](./what-is-this.md) — 한 문장 소개와 쓰는 맥락, 그리고 안 맞는 경우까지. 3분이면 정체가 잡혀요.
2. [`5분 투어`](./five-minute-tour.md) — 모듈 그림과 함께 핵심 개념 네 가지를 훑어요. "아하" 가 오는 지점이에요.

여기서 마음이 기울었다면, 손에 잡고 따라 하는 세 편으로 이어 가요.

3. [`첫 실행 결과 해석`](./first-run.md) — `bootRun` 로그를 한 줄씩 읽어 무슨 일이 일어나는지 파악해요.
4. [`첫 수정`](./first-change.md) — `nickname` 컬럼 하나를 더하며, 한 필드가 코드의 여섯 곳에 어떻게 반영되는지 따라가요.
5. [`배포 맛보기`](./first-deploy.md) — 로컬 Docker 로 운영 배포의 감을 잡아요.

> 💡 읽다가 모르는 용어가 나오면 [`Level 0 용어 사전`](../reference/glossary.md) 에서 먼저 찾아보세요. Spring·JPA·Docker·JWT·Kamal 같은 범용 용어를 직관적으로 풀어 뒀어요.
